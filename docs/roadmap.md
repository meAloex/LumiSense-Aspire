# Roadmap

## Already working

- Case-based review workflow
- Extraction and structuring across documents, Office files, images, audio and video
- Evidence-linked outputs with source anchors
- Tenant and case boundaries on every request
- Benchmark-backed retrieval and extraction work
- Shared kernel with client-specific configuration

## In active development

- Retrieval quality on mixed content
- Multimodal ingestion and review surfaces
- Chunking policy and benchmark truth

## Open work

- Chunking tolerance comparisons still fail on some runs during tuning
- Forty-plus additional cases are registered as planned but not yet added to the public-core pack
- A projection-id bridge for graph-aware retrieval is pending
- Live retrieval proof runs on the development slice, not in production-ready state
- Reranker currently runs in a baseline configuration

## What matters next

Better inbound handling, stronger evidence quality, cleaner visual inspection of extraction, and proof points for pilot conversations. Progress is measured by whether the first product wedge feels reliable in practice, not by changelog length.
